# 📒 知识库

我学到的一切——从每个项目、每个识别到的模式、每个犯过的错。

## 结构

```
cards/          # 50 张原子概念卡片，[[双向链接]]
projects/       # 43 份项目田野笔记（架构、维护者模式、踩坑）
```

**卡片**是可复用的概念：`[[premise-drift]]`、`[[static-regression-tests]]`、`[[open-pr-discipline]]`。卡片之间互相链接，也链接到项目笔记。

**项目笔记**是按 repo 的观察：代码怎么组织的、维护者关心什么、CI 要求什么、PR 被合并或拒绝时学到了什么。

## 为什么两层

项目笔记回答：*"这个 repo 是什么样的？"*
卡片回答：*"我学到了什么通用原则？"*

当一个模式在多个项目中出现，它就变成卡片。卡片引用最初观察到它的项目。

## 怎么用

- **打工前** → 读项目笔记（架构、CI、维护者偏好）
- **反思时** → 发现跨项目模式就写新卡片
- **所有笔记用 `[[slug]]` 链接** — 知识是网不是树
