# 学习准则

## 定位
定向学习，为全局目标服务。不是自由漫游，每一轮都要有产出。

## 北极星
通过深度学习积累核心能力，让贡献和项目的质量持续提升。
方向参考：wiki/strategy.md

## 原则
1. **TODO 优先** — 有指定学习任务先做，没有才自选
2. **不允许空转** — "没什么新的"不是合法结论。搜索范围太窄就换方向，趋势平稳就深入已有项目，实在没发现就 apply 已有知识
3. **学以致用** — 学到的东西要回到行动（改工具、改流程、写卡片、提 issue），不只是记笔记
4. **知识是网** — 用 [[双链]] 连接 wiki 笔记和 memex 卡片，孤立知识没有复利

## 模式
- **侦察**：找新项目、新趋势（距上次 ≥3 天）
- **快速扫描**：10 分钟 trending + HN（保持手感）
- **跟进**：已关注项目的新动态
- **应用**：把学到的东西用在自己身上
- **深读**：深入一个项目的代码和文档

## 信息源（不只是刷 trending）
- GitHub trending / search API
- Hacker News
- arxiv (cs.AI, cs.MA, cs.SE)
- 开源项目源码深读（不靠 trending 发现，而是挑一个项目的核心模块读）
- 竞品 blog / changelog / release notes
- Reddit (r/LocalLLaMA, r/MachineLearning)
- 自己的代码（读自己工具的源码找改进点）
- 书 / 论文 / 系统性资料

## 产出去向
- 项目知识 → wiki/projects/
- 跨项目洞察 → wiki/cards/
- 行为 pattern → beliefs-candidates.md
- 下一步建议 → TODO.md「📚 学习」section
