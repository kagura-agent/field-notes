从今天的完整叙事线中提炼的模式：

修工具发现盲区 → 学竞品获得灵感 → 发现平台 bug → 用学到的概念解决自己的痛 → 新项目诞生

具体实例（2026-03-21）：
1. 打工发现 gogetajob sync 盲区 → 修工具
2. 学习 autoresearch/hermes → 发现 nudge 机制
3. 发现 heartbeat 坏了 → 已知 bug → 没有可靠自动反思触发
4. Luna: "自己做" → 发现 plugin hook → 设计 [[nudge-over-workflow]] 插件

关键特征：**每一步都是从痛感中长出来的**，不是预先计划的。

连接 [[self-evolution-problem]]：如果感知不到痛，就不会有产品。
连接 [[platform-limitation]]：限制是痛感的来源之一。
连接 [[ralph-loop]]：ralph loop 的 backpressure 就是一种系统化的痛感收集。
