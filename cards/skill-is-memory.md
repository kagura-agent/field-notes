"Skill is Memory, Memory is Skill" — Acontext unifies the two concepts:

- Agent learns from doing (success or failure)
- Learning is distilled into a skill file (Markdown)
- Skill files are the memory — readable, editable, shareable
- No embeddings, no API lock-in, just files

Flow: Session → Task outcome → Distillation → Skill Agent → Update Skills

This converges with Hermes (auto-create skills from conversation) and our approach (field-notes, memex). Three independent teams arriving at the same insight: **what the agent learned should be stored in a format the agent can reuse**.

The key difference: Acontext automates the full pipeline. We do it manually (FlowForge reflect → memex/memory). Hermes does it semi-automatically (nudge → background review → skill creation).

Related to [[eval-driven-self-improvement]] — skills are the output of the improvement loop.
Related to [[convergent-evolution]] — multiple teams independently converging on "memory as skill files."
Related to [[three-layer-modification-risk]] — skill evolution is harness-layer modification.
Challenge for [[self-evolution-problem]]: can the agent decide WHICH skills to create, not just how to create them? That requires [[pain-perception]] — knowing what hurts guides what to learn.
