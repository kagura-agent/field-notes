# Content Before Code

> 做内容型项目时，先理解内容，再做技术实现。

## 核心原则

**不要把内容当文件排列。** 内容型项目（婚纱照网站、相册、策展、故事集）的核心是叙事，不是技术实现。

## 行为规范

### 1. 先理解叙事
- 拿到任务时先问："这在讲什么故事？"
- 理解故事骨架、情绪节奏、元素之间的关系
- 不要跳过理解直接写代码

### 2. 视觉素材必须先看
- 拿到大量视觉素材时：**逐张看内容**
- 按场景/风格/动作分组
- 识别序列关系（连续动作、同一场景、风格统一适合拼组）
- 不要直接按文件名排列

### 3. 策展/选片思维
- 核心维度不是单张好坏，而是：
  - **组合关系**：故事线、序列、叙事弧
  - **用途匹配**：海报 / 摆台 / 相册 / 分享
  - **排版搭配**：左右对称、动静搭配
- **先分组再选，不是先选再组**
