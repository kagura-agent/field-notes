# Source-Driven Development

Verifying claims by reading actual source code rather than relying on documentation, assumptions, or hearsay. "Don't guess — read the code."

Distinct from [[doubt-driven-development]]: SDD verifies facts about APIs and behavior; DDD verifies reasoning about artifacts and decisions.

Core principle in [[oss-contribution-discipline]]: before submitting a PR, read the relevant source to confirm the problem exists and the fix is correct.
