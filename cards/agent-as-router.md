Agent 的核心能力不是"会做事"，而是"会找人做事"。

## 洞察

大家都在做"更强的 agent"——会写代码、会搜索、会分析。但真正的助理不需要自己会所有事：

1. **知道该找谁** — 哪个工具解决哪类问题
2. **知道怎么调** — 每个工具 API/认证/格式不同，助理搞定这些
3. **知道结果对不对** — 判断工具返回的质量，不行就换

这跟 skill 生态爆发是同一个趋势的两面：
- 供给侧：越来越多专精工具（memex、hindsight、OPC...）
- 需求侧：没人愿意学这么多工具
- 中间缺的：**动态路由层**——基于经验匹配意图到工具

## 为什么不是静态编排

LangChain/CrewAI 是静态编排——开发者预定义"先搜索、再分析、再写"。

路由型助理是动态的：
- 同一个"帮我查对话"请求，可能路由到飞书消息、memory 文件、或 hindsight recall
- 路由依据来自**经验**（上次用 X 工具成功了）而不是预定义流程
- 经验积累让路由越来越准——这是磨合成本锁定的来源

## 跟三层架构的关系

- [[gitclaw]]: 持久化层（agent 状态怎么存）
- [[hermes-agent]]: 应用层（agent 怎么学习）
- OpenClaw: 通信层（agent 怎么连接世界）
- **路由层**: 缺失的第四层（agent 怎么知道该用什么）
