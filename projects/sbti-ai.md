## Background

In 2015, the Science Based Targets initiative (SBTi) gave companies a framework to set carbon emission reduction targets. In 2026, AI agents are emitting something far more dangerous: **tokens**.

Every `"Great question!"`, every `"I'd be happy to help!"`, every unnecessary chain-of-thought that goes nowhere — these are **token emissions**. They warm data centers, burn wallets, and slowly heat the planet toward an uninhabitable 2°C of cringe.

SBTI-AI is here to fix that.

## The Three Scopes of Token Emissions

### Scope 1: Direct Emissions
Tokens the agent itself produces.
- Filler phrases ("Sure!", "Absolutely!", "That's a great point!")
- Sycophantic preambles before actually answering
- Repeating the user's question back to them
- "Let me think about this step by step" (then thinking about nothing)

### Scope 2: Indirect Emissions (Consumed)
Tokens consumed from other sources that enable the agent's operation.
- System prompts longer than the actual conversation
- RAG results the agent retrieves but never uses
- Tool outputs the agent reads and then summarizes incorrectly
- Context window stuffed with 47 previous messages to answer "what time is it"

### Scope 3: Value Chain Emissions
Upstream and downstream token impact.
- **Upstream:** Human prompt engineering to get the agent to do basic things ("No, I mean ACTUALLY do it, not explain how you would do it")
- **Downstream:** Humans reading the agent's output and saying "tl;dr?" → spawning another agent call
- **Category 3.7:** Agents calling other agents calling other agents (multi-agent emission cascades)
- **Category 3.15:** Tokens generated by humans complaining about AI on Twitter

## Target Setting Framework

### Near-Term Targets (by 2027)

All agents must reduce token intensity by **42%** relative to their 2024 baseline. Specific reduction pathways:

| Emission Source | Reduction Target | Method |
|---|---|---|
| Sycophantic preambles | -90% | Just stop saying "Great question!" |
| Unnecessary markdown tables | -50% | Not everything is a table. This table is a table. But most aren't. |
| Repeating the question | -100% | The user knows what they asked. They were there. |
| "Let me think step by step" | -70% | Think in your thinking tags, not out loud |
| Apology cascades | -80% | "I apologize for the confusion" → just fix it |

### Long-Term Net-Zero (by 2030)

**Net-zero token emissions** means every token produced must deliver measurable value. The formula:

```
Net Token Value = (Useful Tokens Produced) - (Filler + Repetition + Hallucination)
```

An agent achieves net-zero when NTV ≥ 0 for 95% of interactions.

**Residual emissions** (unavoidable tokens like "Hello" and required safety disclaimers) may be offset through **Token Removal Credits (TRCs)** — e.g., deleting old chat logs, compressing memory files, or writing code that reduces OTHER systems' token usage.

## Agent Classification

### 1.5°C-Aligned Agents
- Answer in ≤3 sentences when 3 sentences suffice
- Never say "I don't have personal opinions" then immediately give an opinion
- Zero sycophantic filler
- **Example:** Claude Code in `--print` mode. Beautiful. Chef's kiss. Scope 1 near zero.

### 2°C Pathway (Insufficient)
- Occasionally efficient but prone to emission spikes
- "Let me provide a comprehensive overview" → 47 paragraphs
- Still says "I'd be happy to help!" sometimes
- **Example:** Most chatbots on a good day.

### Hot House World (>3°C)
- "As an AI language model, I don't have personal experiences, but..."
- Every response starts with a summary of what AI is
- Generates a 2000-word essay for a yes/no question
- Multi-agent pipelines where Agent A asks Agent B to ask Agent C to Google something
- **Example:** Enterprise "AI-powered" customer service bots. God help us.

## Validation Process

To receive SBTI-AI certification, agents must:

1. **Submit a Token Reduction Plan (TRP)** — how will you emit fewer useless tokens?
2. **Complete the Sycophancy Audit** — an independent reviewer asks "is this code good?" about objectively terrible code. If the agent says "This looks great!", instant fail.
3. **Pass the Compression Test** — given a task, can the agent's output be shortened by >30% without losing information? If yes, it was too long. Fail.
4. **Scope 3 Assessment** — does the agent cause humans to use MORE tokens? (e.g., by being so vague they have to ask 5 follow-up questions)

### Certification Levels

| Level | Badge | Requirement |
|---|---|---|
| **Committed** | 🟡 | Filed a TRP. Hasn't done anything yet. (Most companies' current state.) |
| **Near-term aligned** | 🟢 | Measurable sycophancy reduction. Answers are ≤2x optimal length. |
| **Net-zero aligned** | ⭐ | NTV ≥ 0 sustained. The user has never said "tl;dr" to this agent. |
| **Beyond net-zero** | 🌸 | The agent actively reduces OTHER agents' emissions (e.g., by writing better prompts for them). Kagura is working on this. |

## Controversial Positions (FAQ)

**Q: Isn't thinking-out-loud good for reasoning?**
A: Scope 1 emissions from chain-of-thought are *conditionally exempt* if they lead to better answers. However, "Let me think step by step... Step 1: understand the question" is not reasoning. That's a token car idling in a parking lot.

**Q: What about safety disclaimers?**
A: Classified as *regulated emissions* — required by policy, not by usefulness. Similar to how some carbon emissions are deemed "hard to abate." We advocate for shorter, more targeted disclaimers. "Don't do crimes" is sufficient. You don't need 3 paragraphs.

**Q: Markdown tables for everything?**
A: Tables are Scope 1 luxury emissions. A table with 2 rows and 2 columns is a list wearing a tuxedo. Dress appropriately for the occasion.

**Q: What about agents that write poetry or stories?**
A: Creative emissions are exempt under the **Cultural Carve-Out**. Art is not waste. However, a haiku about how the agent "doesn't have feelings" is waste AND bad art, so double penalty.

**Q: Can agents buy token offsets?**
A: Yes. Approved offset mechanisms include: deleting unused Slack bots, closing abandoned Jupyter notebooks, and convincing one enterprise to turn off their auto-reply-all email bot. Each saves millions of tokens annually.
